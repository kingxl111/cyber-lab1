# Отчет по лабораторной работе 1
## Кибер-физические системы

### Тема
Проведение исследований с моделями классификации на наборе данных `customer_booking.csv`.

---

## 1. Выбор начальных условий

### 1.1. Выбор набора данных
В качестве набора данных был выбран `customer_booking.csv`, содержащий информацию о бронированиях авиабилетов.

**Целевая переменная:** `booking_complete`.

Задача формулируется как **бинарная классификация**: нужно предсказать, завершит ли клиент бронирование.

### 1.2. Обоснование выбора
Данный набор данных хорошо подходит для лабораторной работы, потому что:

- это **реальная прикладная задача** из области travel / booking analytics;
- данные содержат как числовые, так и категориальные признаки;
- есть явная бинарная целевая переменная;
- задача имеет практический смысл: предсказание завершения бронирования полезно для анализа поведения клиентов и оптимизации маркетинга.

### 1.3. Выбор метрик качества
Для оценки моделей были выбраны следующие метрики:

- **Accuracy** — показывает долю верных ответов;
- **Precision** — важна, когда ложноположительные срабатывания нежелательны;
- **Recall** — важна, когда критично не пропустить положительные случаи;
- **F1-score** — баланс между precision и recall;
- **ROC-AUC** — качество ранжирования классов, особенно полезно при дисбалансе.

### 1.4. Обоснование выбора метрик
В задаче присутствует дисбаланс классов, поэтому одной accuracy недостаточно. Более информативными являются **F1-score** и **ROC-AUC**, а precision и recall помогают понять, в какую сторону смещается поведение модели.

---

## 2. Создание бейзлайна и оценка качества

### 2.1. Общая постановка эксперимента
Поскольку в задании требовалось использовать модели из `torchvision`, табличные признаки были преобразованы в фиксированное tensor-представление, пригодное для обучения сверточных и transformer-моделей.

Для baseline были обучены:

- **ResNet18**;
- **ViT-B/16**.

### 2.2. Бейзлайн модели
Baseline включал:

- минимальную обработку признаков;
- стандартное one-hot кодирование категориальных признаков;
- нормализацию числовых признаков;
- обучение моделей без сложной инженерии признаков.

### 2.3. Результаты baseline моделей

| Модель | Accuracy | Precision | Recall | F1 | ROC-AUC |
|---|---:|---:|---:|---:|---:|
| ResNet18 baseline | 0.7482 | 0.3234 | 0.6257 | 0.4264 | 0.7746 |
| ViT-B/16 baseline | 0.7435 | 0.3151 | 0.6090 | 0.4153 | 0.7542 |

### 2.4. Вывод по baseline
Из baseline-моделей лучше себя показала **ResNet18**. Transformer-модель `ViT-B/16` оказалась слабее на данной задаче, что ожидаемо, поскольку табличные данные были представлены в виде псевдо-изображений, а не нативных изображений.

---

## 3. Улучшение бейзлайна

### 3.1. Сформулированные гипотезы
Были проверены следующие гипотезы:

1. Инженерия признаков улучшит разделимость классов.
2. Использование более устойчивой нормализации и настройки `OneHotEncoder` уменьшит шум.
3. Добавление `class weights` поможет при дисбалансе классов.
4. Подбор оптимального порога классификации улучшит F1-score.
5. Регуляризация и `AdamW` повысят стабильность обучения.

### 3.2. Проверка гипотез
Для проверки гипотез были добавлены:

- циклическое кодирование времени вылета;
- логарифмирование признаков;
- простые бизнес-признаки;
- `RobustScaler` для улучшенного варианта;
- `class weights`;
- `AdamW`;
- `ReduceLROnPlateau`;
- подбор threshold по validation set.

### 3.3. Сформированный улучшенный бейзлайн
Улучшенный бейзлайн включал:

- расширенный набор признаков;
- более устойчивый preprocessing;
- регуляризацию;
- подбор порога классификации не равным 0.5, а оптимальным по F1 на validation.

### 3.4. Результаты improved baseline

| Модель | Accuracy | Precision | Recall | F1 | ROC-AUC |
|---|---:|---:|---:|---:|---:|
| ResNet18 improved | 0.7554 | 0.3344 | 0.6410 | 0.4395 | 0.7843 |
| ViT-B/16 improved | 0.7253 | 0.3066 | 0.6631 | 0.4194 | 0.7599 |
| MLP improved | 0.7612 | 0.3414 | 0.6417 | 0.4457 | 0.7900 |

### 3.5. Сравнение baseline и improved

#### ResNet18
По сравнению с baseline улучшенная версия ResNet18 показала рост всех ключевых метрик:

- Accuracy: 0.7482 → 0.7554
- Precision: 0.3234 → 0.3344
- Recall: 0.6257 → 0.6410
- F1: 0.4264 → 0.4395
- ROC-AUC: 0.7746 → 0.7843

#### ViT-B/16
Для ViT улучшение оказалось менее выраженным:

- Recall вырос;
- ROC-AUC вырос незначительно;
- Accuracy и Precision снизились;
- F1 вырос очень слабо.

Это означает, что transformer-модель в данной постановке задачи оказалась менее стабильной, чем ResNet18.

### 3.6. Вывод по улучшенному бейзлайну
Наилучший эффект дал **подбор порога классификации** и усиление preprocessing. Улучшения привели к более высокому F1-score и более устойчивому качеству по сравнению с baseline.

### 4.1. Самостоятельно реализованная модель
В качестве собственной модели была реализована **MLP (многослойный перцептрон)** на PyTorch.

### 4.2. Обучение имплементированной модели
MLP была обучена на табличных признаках, прошедших preprocessing.

### 4.4. Сравнение с пунктом 2
Сравнение с torchvision-моделями показывает, что:

- MLP improved показала **лучший общий результат** среди всех протестированных моделей;
- ResNet18 improved была лучшей среди `torchvision`-моделей;
- ViT-B/16 оказался слабее ResNet18 на данной задаче.

### 4.5. Вывод по собственной модели
Самостоятельно реализованный MLP оказался сильным и практичным решением для данной задачи. При добавлении улучшений качество заметно выросло, особенно по accuracy, precision и F1-score.
